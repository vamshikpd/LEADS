Radial Search

 

The radial search functionality enables the user to perform a location based search. The location can be a city name, an address, intersection of two streets etc.

 

 

The radial search pop-up provides the user with the following input fields:

  • Location OR Intersecting Streets*
  • Zip Code
  • Search Radius*

(*) indicates mandatory fields

 

When the user enters a location or intersecting streets along with the search radius and clicks the search button, LEADS2.0 performs a radial search. This search takes the location as the center point and constructs a circle around the center where the radius of the circle is the search radius specified by the user. The parolees whose current addresses fall within this circle are returned in the search results.

 

If user is entering a location, it recommended specifying the city and state as well. For example, specifying the location as “2nd street” may lead to unexpected results as the mapping functionality would find multiple matches for this address. A better location to specify would be “2nd street, Sacramento, CA” as this would return a unique match for the address location.  Another alternative would be to specify the location as “2nd street” and zip code as “95814”. This would result in a unique match for location too.

 

Example 1:

Location: Folsom, CA

Radius: 5 miles

A radial search would be performed with Folsom, CA as the center and a search radius of 5 miles.

 

Example 2:

Intersection street 1: Sunrise Blvd

Intersection street 2: Folsom Blvd

Radius: 15 miles

A radial search would be performed with the intersection of Sunrise Blvd and Folsom Blvd as the center and a search radius of 15 miles.

 

Example 3:

Location: Sacramento

Zip code: 95825‎

Radius: 5 miles

A radial search would be performed with Sacramento, 95825 as the center and a search radius of 5 miles.
