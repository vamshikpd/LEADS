Step 1 – Case#/Reason (required)

When you enter the main Search page for the first time, the Case# and Reason dialog pop-up box automatically appears over the Search page.

The Help icon is displayed in the bottom right hand corner of the Case#/Reason box.

Click the help icon so you can view Help documentation related to the Case#/Reason function or the entire LEADS2 application.

The Case#/Reason pop-up box requires you to type a Case # and select a Reason for your search. Both fields are used for auditing activities in the LEADS2 application.

Both data fields are required.

If you don’t enter data into both fields, the following error message displays: Error: “Please enter Case # and select Reason for Search”.

 

Type your Case #.

If you don’t enter data a Case#, the following error message displays: Error: “Please enter Case#”.

Example

1234

Press the <Tab> key (on the left side of your keyboard) to move to the next field or click your mouse in the next field.

Select the Reason for your search from the drop-down list.

You can type or select the following reasons from the list:

•            Criminal Investigation

•            Crime Analysis

•            Suspect ID

Type the first character of the reason and the first word matching that letter is displayed.

 

Example

Type the letter s or S and the reason Suspect ID is displayed. Type the letter c or C and Criminal Investigation will appear with Crime Analysis listed below it.

Press the <Enter> key and the ‘Matching Records: #,###’ message is displayed on the Parolee Results action bar along with a list of  all parolees in the LEADS2 application database.

The pound signs (##) represent the total number of parolees listed on your screen.

If you don’t select a Reason for your search, the following error message displays: Error: “Please select Reason for Search”.
